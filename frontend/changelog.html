Changelog

Apr 1-3, 2026

Security Performance Reliability
Full platform overhaul
  • Every layer of Vidia was audited, restructured, and hardened: the backend, the frontend, the GPU worker, and the video processing pipeline. This is the largest single update since launch.
Tighter security
  • Authentication is now enforced on all sensitive endpoints. Generating a video requires a valid login.
  • All communication between the GPU worker and the backend is cryptographically verified.
  • Rate limiting protects login, upload, and generation from abuse.
Faster loads
  • The app now bundles and minifies all code before deployment. Page load dropped from 30+ network requests to 3, with automatic cache-busting so you always get the latest version.
More reliable results
  • Your video now appears on screen as soon as it is ready, and the notification email sends correctly.
  • False error messages ("generation heartbeat lost") no longer appear after a successful generation.
  • Starting a new generation properly clears the previous result.
Under the hood
  • The backend was rebuilt from a single 1,500-line file into a modular architecture with dedicated routing, middleware, and utilities.
  • The video processing node was split into six focused modules and now auto-updates on every GPU worker restart.
  • All frontend feature modules follow a consistent structure, making it faster to add new features going forward.
  • A staging environment is available for testing changes before they reach production.
  • Multiple long-standing bugs fixed across function signatures, state management, and mode detection.
